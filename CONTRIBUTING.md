# Contributing to Artist Portfolio

## How to Contribute

We welcome contributions to improve this artist portfolio project!

### Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature
4. Make your changes
5. Commit your changes with a clear message
6. Push to your fork
7. Create a Pull Request

### Code Standards

- Use semantic HTML5 elements
- Follow CSS Grid best practices
- Ensure mobile-first responsive design
- Add comments for complex code
- Test on multiple devices and browsers

### Branch Naming

- `feature/` for new features
- `bugfix/` for bug fixes
- `docs/` for documentation

### Commit Messages

Use clear, descriptive commit messages:
- Bad: "Update stuff"
- Good: "Add responsive grid layout for tablet screens"

### Pull Request Process

1. Update the README if needed
2. Test your changes thoroughly
3. Ensure code follows our standards
4. Request review from team members
5. Address feedback and iterate
6. Merge when approved
